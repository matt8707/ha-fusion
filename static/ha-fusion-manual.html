IntroduzioneIntroduction

ha-fusion è una dashboard personalizzata per Home Assistant, costruita con SvelteKit. Sostituisce l'interfaccia predefinita di HA con un'esperienza moderna, fluida e completamente configurabile tramite interfaccia grafica — niente YAML da scrivere a mano per la maggior parte delle funzioni.

ha-fusion is a custom dashboard for Home Assistant, built with SvelteKit. It replaces the default HA interface with a modern, fluid, fully configurable experience — no manual YAML required for most features.

Questo è un fork personale del progetto originale ha-fusion di matt8707, esteso con funzionalità aggiuntive tra cui il nuovo tipo di elemento Custom Panel, nuovi temi, CSS personalizzato in-app e miglioramenti vari.

This is a personal fork of the original ha-fusion by matt8707, extended with additional features including the new Custom Panel item type, new themes, in-app custom CSS, and various improvements.

Installazione — Add-on Home AssistantInstallation — Home Assistant Add-on

Il metodo più semplice per chi usa Home Assistant OS o Home Assistant Supervised. L'add-on viene gestito direttamente dal Supervisore HA.

The easiest method for users running Home Assistant OS or Home Assistant Supervised. The add-on is managed directly by the HA Supervisor.

PassiSteps

  1. Aggiungi il repository — Vai in Impostazioni → Add-on → Store degli add-on → ⋮ → Repository e aggiungi:
    https://github.com/amedello/addon-ha-fusion
  2. Add the repository — Go to Settings → Add-ons → Add-on Store → ⋮ → Repositories and add:
    https://github.com/amedello/addon-ha-fusion
  3. Installa — Aggiorna la pagina dello Store, cerca ha-fusion e clicca Installa.
  4. Install — Refresh the Store page, search for ha-fusion and click Install.
  5. Avvia — Clicca Avvia nella pagina dell'add-on. Attiva Mostra nella barra laterale per accesso rapido.
  6. Start — Click Start on the add-on page. Enable Show in sidebar for quick access.
  7. Accedi — Apri la WebUI. La prima volta dovrai autenticarti con il tuo account Home Assistant.
  8. Access — Open the WebUI. The first time you'll need to authenticate with your Home Assistant account.

Installazione — DockerInstallation — Docker

Usa Docker (o Docker Compose) per installare ha-fusion su qualsiasi macchina Linux, macOS o Windows che abbia accesso di rete alla tua istanza Home Assistant.

Use Docker (or Docker Compose) to install ha-fusion on any Linux, macOS or Windows machine with network access to your Home Assistant instance.

Docker Compose

Crea un file docker-compose.yml:

Create a docker-compose.yml file:

services:
  ha-fusion:
    container_name: ha-fusion
    image: ghcr.io/amedello/ha-fusion
    volumes:
      - /percorso/locale/ha-fusion:/app/data  # dati persistenti
    network_mode: bridge
    ports:
      - "5050:5050"
    environment:
      TZ: Europe/Rome          # fuso orario
      HASS_URL: http://192.168.1.x:8123  # URL di Home Assistant
    restart: always

Avvia il container:

Start the container:

docker-compose up -d ha-fusion

L'interfaccia è disponibile su http://<ip-macchina>:5050

The interface is available at http://<machine-ip>:5050

dashboard.yaml

Il file principale della configurazione si trova in data/dashboard.yaml. Contiene tutte le viste, sezioni, elementi e la configurazione della sidebar. Viene scritto automaticamente quando salvi dalla UI, ma può essere modificato a mano o tramite l'editor YAML integrato.

The main configuration file lives at data/dashboard.yaml. It contains all views, sections, items and sidebar configuration. It is written automatically when you save from the UI, but can also be edited manually or via the built-in YAML editor.

# struttura di base / basic structure
theme: godis          # nome del tema
hide_views: false    # nasconde la barra delle viste
hide_sidebar: false  # nasconde la sidebar
sidebarWidth: 380    # larghezza sidebar in pixel
views:               # array di viste
  - id: 1234567890
    name: Casa
    icon: mdi:home
    sections: # …
sidebar:             # array di elementi sidebar
  - type: time
    id: 9876543210
IDs — Ogni vista, sezione ed elemento deve avere un id numerico univoco nel dashboard. Usa sempre la UI per creare elementi: genera automaticamente gli ID con generateId(). Se modifichi il YAML a mano, genera un numero intero random di 13 cifre.Every view, section and item must have a numerically unique id across the whole dashboard. Always use the UI to create items — it auto-generates IDs via generateId(). If you edit YAML manually, generate a random 13-digit integer.

Panoramica dell'InterfacciaInterface Overview

L'interfaccia di ha-fusion si compone di tre aree principali:

The ha-fusion interface consists of three main areas:

  1. Barra delle viste — in alto, mostra le tab delle viste configurate. Cliccando su una vista si naviga al suo contenuto.
  2. Views bar — at the top, shows tabs for configured views. Clicking a view navigates to its content.
  3. Area principale — la griglia centrale con sezioni ed elementi. Occupa la maggior parte dello schermo.
  4. Main area — the central grid with sections and items. Occupies most of the screen.
  5. Sidebar — a destra, mostra widget informativi (ora, meteo, sensori, ecc.).
  6. Sidebar — on the right, shows informational widgets (time, weather, sensors, etc.).

Icona menu (☰)Menu icon (☰)

In alto a sinistra si trova l'icona del menu. Apre il Drawer che contiene le opzioni di configurazione globale: temi, CSS, lingua, impostazioni e aggiornamento.

In the top left is the menu icon. It opens the Drawer containing global configuration options: themes, CSS, language, settings, and updates.

Pulsante Modifica (✏️)Edit button (✏️)

In alto a destra si trova il pulsante per entrare in modalità modifica. In questa modalità compaiono tutti i controlli per aggiungere, spostare e configurare elementi.

In the top right is the button to enter edit mode. In this mode all controls appear for adding, moving and configuring items.

VisteViews

Le viste sono le tab principali di navigazione. Ogni vista può contenere un numero arbitrario di sezioni. Nel YAML corrispondono all'array views in cima a dashboard.yaml.

Views are the main navigation tabs. Each view can contain any number of sections. In YAML they correspond to the views array at the top of dashboard.yaml.

Aggiungere una vistaAdding a view

In modalità modifica, clicca l'icona + nella barra delle viste. Inserisci nome e icona, poi salva.

In edit mode, click the + icon in the views bar. Enter a name and icon, then save.

Modificare una vistaEditing a view

In modalità modifica, fai clic sul nome della vista per modificarlo; oppure usa il pulsante Configura (ingranaggio) che appare accanto alla tab.

In edit mode, click the view name to edit it; or use the Configure (gear) button that appears next to the tab.

Riordinare le visteReordering views

In modalità modifica, trascina le tab delle viste per cambiarne l'ordine.

In edit mode, drag the view tabs to reorder them.

Nascondere la barra delle visteHiding the views bar

Imposta hide_views: true in dashboard.yaml per nascondere tutta la barra. Utile su schermi piccoli.

Set hide_views: true in dashboard.yaml to hide the entire bar. Useful on small screens.

SezioniSections

Le sezioni organizzano gli elementi all'interno di una vista. Esistono quattro tipi di sezione.

Sections organise items within a view. There are four section types.

Sezione Normale (default)Normal Section (default)

Il tipo base. Mostra un titolo opzionale e una griglia di elementi. Nel YAML non ha il campo type.

The basic type. Shows an optional title and a grid of items. In YAML it has no type field.

sections:
  - id: 1234567890000
    name: "Illuminazione"
    items:
      - type: button
        id: 9876543210001
        entity_id: light.soggiorno

Horizontal Stack

Affianca più sezioni in colonne orizzontali. Ogni colonna è una sezione figlia con i propri elementi. Ideale per dividere lo schermo in più zone affiancate.

Places multiple sections in horizontal columns. Each column is a child section with its own items. Ideal for splitting the screen into side-by-side zones.

- type: horizontal-stack
  id: 1111111111111
  sections:
    - id: 2222222222222
      name: "Sinistra"
      items: # …
    - id: 3333333333333
      name: "Destra"
      items: # …

Vertical Stack Nuovo

Come l'horizontal stack, ma dispone le sezioni figlie in verticale una sopra l'altra. Ogni sezione figlia ha il proprio titolo. Utile per raggruppare zone logicamente correlate in una colonna.

Like the horizontal stack, but arranges child sections vertically one above the other. Each child section has its own title. Useful for grouping logically related zones in a column.

- type: vertical-stack
  id: 4444444444444
  sections:
    - id: 5555555555555
      name: "Piano terra"
      items: # …
    - id: 6666666666666
      name: "Primo piano"
      items: # …
NotaNote — horizontal-stack e vertical-stack non possono essere annidati l'uno nell'altro: uno stack può contenere solo sezioni normali come figli.horizontal-stack and vertical-stack cannot be nested inside each other: a stack may only contain normal sections as children.

A special section optimised for displaying HA scenes. Scene items are shown in a compact grid without a section title.

Elemento: ButtonItem: Button

Il tipo di elemento più versatile. Un button rappresenta qualsiasi entità di Home Assistant: luci, interruttori, script, scene, climate, media player, sensori e molti altri. Al click esegue l'azione predefinita per il dominio (toggle, press, activate…) oppure apre la finestra dettagli dell'entità.

The most versatile item type. A button can represent any Home Assistant entity: lights, switches, scripts, scenes, climate, media players, sensors and many more. On click it performs the default action for the domain (toggle, press, activate…) or opens the entity detail modal.

Esempio baseBasic example

- type: button
  id: 1234567890001
  entity_id: light.soggiorno
  name: "Soggiorno"
  icon: mdi:ceiling-light
  color: "#ffcc44"

Slider luminosità (trascinamento)Brightness slider (drag)

Per le entità light, puoi regolare la luminosità direttamente sulla tile trascinando orizzontalmente. Tieni premuto sulla tile e trascina a destra per aumentare, a sinistra per diminuire la luminosità. Un overlay colorato mostra il livello corrente durante il trascinamento.

For light entities, you can adjust brightness directly on the tile by dragging horizontally. Press and hold on the tile, then drag right to increase or left to decrease brightness. A colour overlay shows the current level during the drag.

Servizio personalizzato (campo service)Custom service (service field)

Quando vuoi che il click esegua un servizio specifico invece del toggle predefinito, usa il campo service. Il valore è una stringa YAML multi-riga con la struttura del servizio.

When you want click to execute a specific service instead of the default toggle, use the service field. The value is a multi-line YAML string with the service structure.

- type: button
  id: 1234567890002
  entity_id: sensor.soundbar
  name: "Soundbar"
  more_info: false
  service: |-
    service: remote.send_command
    data:
      device_id: abc123def456
      command: PowerToggle
      device: Soundbar

- type: button

Custom Panel Nuovo

Custom Panel è un tipo di elemento completamente nuovo che permette di costruire un pannello multi-riga dentro una singola tile. Ogni riga può essere di tipo diverso: telecamera, pulsanti, sensore o slider. Le righe sono riordinabili con drag & drop dalla configurazione.

Custom Panel is a brand-new item type that lets you build a multi-row panel inside a single tile. Each row can be a different type: camera, buttons, sensor or slider. Rows are reorderable via drag & drop in the configuration modal.

Sul tile principale viene mostrato l'icona, il nome e opzionalmente lo stato di una singola entità "primaria" selezionabile dalla configurazione.

The main tile shows the icon, name and optionally the state of a single "primary" entity selected from the configuration.

Struttura YAMLYAML structure

- type: custom_panel
  id: 1234567890020
  name: "Garage"
  icon: mdi:garage
  color: "#4488ff"
  primary_row_id: 10   # id della riga il cui stato appare sul tile
  rows:
    - type: camera
      id: 1
      entity_id: camera.garage
      stream: true
    - type: buttons
      id: 2
      columns: 3
      items:
        - id: 100
          entity_id: light.garage_zona_a
          name: "Zona A"
          icon: mdi:lightbulb
        - id: 101
          entity_id: switch.presa_garage
          name: "Presa"
    - type: sensor
      id: 10
      entity_id: sensor.temperatura_garage
      name: "Temperatura"
      suffix: "°C"
    - type: slider
      id: 20
      entity_id: light.garage_zona_a
      name: "Luminosità"

Condizioni di VisibilitàVisibility Conditions

ha-fusion supporta tre livelli di visibilità condizionale che permettono di mostrare o nascondere parti della dashboard in base agli stati di Home Assistant, valori numerici o dimensioni dello schermo.

ha-fusion supports three levels of conditional visibility that let you show or hide parts of the dashboard based on HA states, numeric values or screen sizes.

1. Visibilità di sezione1. Section visibility

Imposta condizioni sulla sezione stessa. La sezione (e tutti i suoi elementi) viene nascosta quando le condizioni non sono soddisfatte.

Set conditions on the section itself. The section (and all its items) is hidden when conditions are not met.

Configurabile tramite il pulsante occhio (👁) nell'intestazione della sezione in modalità modifica.

Configurable via the eye (👁) button in the section header in edit mode.

2. Visibilità di singolo elemento2. Single item visibility

Condizioni su un singolo elemento. Il solo elemento viene nascosto quando le condizioni non sono soddisfatte, gli altri elementi della sezione restano visibili.

Conditions on a single item. Only that item is hidden when conditions fail; other items in the section remain visible.

Configurabile tramite il pulsante occhio nella finestra di configurazione dell'elemento (modalità modifica).

Configurable via the eye button in the item configuration modal (edit mode).

3. Template visibilità elementi3. Item visibility template Nuovo

Una condizione di visibilità definita sulla sezione che viene applicata automaticamente a tutti gli elementi della sezione. Il placeholder speciale {item.entity_id} viene sostituito con l'entity_id di ciascun elemento prima di valutare la condizione.

A visibility condition defined on the section that is automatically applied to all items in the section. The special placeholder {item.entity_id} is replaced with each item's entity_id before evaluating the condition.

Configurabile tramite il pulsante scansione-occhio nell'intestazione della sezione.

Configurable via the scan-eye button in the section header.

# Esempio: nascondi tutti gli elementi la cui entità è "unavailable"

  - conditions:
    - condition: state
      entity: "{item.entity_id}"   # placeholder sostituito per ogni item
      state_not: unavailable
